# Postcards — UX Upgrade Backlog

## Where the UX stands today

**Strengths.** The core loop is already coherent and honest: the Map screen pairs a shaded-countries/visited-dots map with a live "Cities in view — most people first" list whose rows carry a real one-tap +/✓ toggle (`MapScreen.tsx`), and search already refuses to invent places (`PlaceSearch.tsx`). Stats shows the twin headline units (%-of-world, per-country city/region coverage) with an accessible `Bar` progressbar (`StatsView.tsx`), and Backup/Attribution deliver the local-first, provenance-first promise (`Backup.tsx`, `Attribution.tsx`). The three-tab shell is simple, keyboard shortcuts exist, and the light theme is calm rather than dashboard-y.

**Weaknesses (all verified in code).**
- **Feedback & data-safety gaps.** Every mutation is silent, and `Backup.onImport` calls `setAll` → `db.replaceAllVisits` immediately (`Backup.tsx:44`, `useVisits.ts:73-76`): one tap on a stale file irreversibly wipes the whole log with only passive grey copy as a guard. There is no undo anywhere.
- **Map ergonomics fight the feature.** `.map-screen` is `overflow-y:auto` (`styles.css:141`), so scrolling the list that *describes the current viewport* pushes the map off-screen. Country polygons are shaded but not tappable (no click handler in `MapView.tsx`), so the headline unit (% of world) can't be acted on from the map.
- **Keyboard-first is promised but incomplete.** The search input claims `role="combobox"` with `aria-controls`/`aria-autocomplete` but wires **no** arrow/Enter/Escape handling, and `aria-selected` is misused to mean "visited" (`PlaceSearch.tsx:39-54`) — a real WCAG 4.1.2 mismatch. The spec's quick-add shortcut (US5 AC1) doesn't exist; existing tab shortcuts are invisible; tab changes never move focus.
- **Honesty of numbers.** "% of the world" shows a bare percent with an invisible base (250 ISO entries incl. territories). Stats can show "100% of cities" when the user visited all 3 cities the starter set knows, and renders empty 0-value bars next to "dataset not loaded".
- **Dead/disconnected surfaces.** The topbar is pure brand chrome, the three tabs are islands (a visited-city row in `VisitsList.tsx:34` even *looks* tappable but is inert), the Places tab stacks three unrelated jobs, and the thin starter dataset can read as "broken app" when panning into an uncovered region.

The backlog below fixes the honest/safety gaps first (cheap, high leverage), then the map-loop and a11y wins, then the structural bigger bets.

## Top quick wins (do first — S effort, high/medium impact)

These are the cheapest, highest-leverage items; most touch only `styles.css` or a single component and several fix genuine honesty/safety/a11y defects.

1. **Pin the map, scroll only the list (#1).** Change `.map-screen` to `overflow:hidden` and give `.view-list` `flex:1; min-height:0; overflow-y:auto`. Pure CSS, no JS, and it un-breaks the feature's core premise (the map staying visible while you read the viewport list). Highest ratio in the whole backlog.
2. **Gate Import behind a counted confirm (#2).** Before `setAll`, if `visits.length>0` show an inline "Replace your N places with the M in this file? [Replace] [Cancel]". Turns a silent, unrecoverable full-wipe into an informed choice; reuses existing `.notice`/`.btn` styles.
3. **Show the denominator on "% of the world" (#6)** and **honest per-country denominators / no empty bars (#7).** `computeCoverage`/`computeCountryCoverage` already return the counts; render `24 / 250` and "of N known cities", and skip the `<Bar value={0}>` when `total===0`. Directly strengthens the "aggregator, never an author" constitution principle at near-zero cost.
4. **Denser city rows with right-aligned compact population (#8)** and **live count + "top 30 of K" cap notice (#9).** Surfaces the actual sort key, fits more rows above the fold, and tells the user the list is capped (so a missing city reads as "zoom in", not "broken") — the key scaling-honesty fix for when the full gazetteer lands.
5. **Sort "By country" by coverage (#11).** Replace the hard `localeCompare` in `visitedCountriesList` with an optional sort key + a native `<select>`; makes the list tell a story and scales as the country count grows.
6. **Announce "no matches" to SR (#12), fix Backup labels + success color (#13), and move focus/announce section on tab change (#14).** Three cheap correctness fixes for stated non-negotiables: closes a WCAG feedback gap on the primary add path, stops "Export map (.md)" from lying (it exports a list), and makes tab changes coherent for AT (`aria-current="page"` + focus move + polite live region).
7. **Add the "/" quick-add shortcut + "?" shortcuts overlay (#15).** Closes spec US5 AC1 (currently unimplemented) and makes the existing invisible tab shortcuts learnable. Use a Zustand focus-nonce (not a CustomEvent) since `MapScreen`/`PlaceSearch` are lazy-mounted, and guard against auto-focus-on-mount popping the mobile keyboard.

## Bigger bets

- **Reframe "Places" as a browsable country checklist (#16, L/High).** Render the already-loaded ~249-country catalog with the same +/✓ toggle used on the map, plus a text filter and an All/Visited segmented control. This is the canonical "how many countries have I been to" interaction and removes today's constraint that the *only* way to mark a country is typing into search. Preserve the existing rich visited list (cities + date/note + Remove) as the "Visited" view rather than replacing it, and plan for grouping cities under countries + virtualization when the gazetteer lands.
- **Close the map loop (#4 + #17 + #21, and later #37).** Make country polygons toggle visited on tap (#4) — the single biggest map win, acting on the headline unit in place. Then draw the in-view unvisited cities as hollow dots from the same list the panel shows (#17) so list and map share one mental model, add "Fit to my places" + tap-country-to-zoom viewport jumps (#21), and once the source carries many dots, add clustering (#37). **Coordination note:** #4 (toggle) and the zoom-on-country-tap half of #21 both want the `countries-base` click; let a country *tap* toggle visited and put zoom-to-country behind a distinct affordance (or long-press/secondary), so one gesture isn't overloaded.
- **Data-safety spine: Toast + Undo (#3).** A tiny dependency-free toast store with a focusable Undo, snapshotting `visits` before each mutation and restoring via `setAll`, wired at all four call sites. Complements the import confirm (#2) as belt-and-suspenders and gives the currently-silent add/remove actions real confirmation. Keep it one restrained plain toast (respect the existing `prefers-reduced-motion` block) to honor "no BS / not AI-looking".
- **A11y depth: keyboard combobox (#5) + command palette (#27) + reflow (#28).** #5 is the load-bearing fix (real ARIA mismatch on the primary control); #27 and #28 round out keyboard-first + WCAG 2.1 AA for desktop/PWA and large-text users. Build the palette lean (dialog + filter + listbox + ~6 actions incl. a "Show shortcuts" entry) and note it only benefits desktop keyboard users on this mobile-first app.
- **Stats depth (#18 + #19).** Continent-coverage bars (Europe 6/50, Asia 2/50 …) give a satisfying mid-level "how much of each part of the world" read, and native-`<details>` per-country drill-down answers the natural "which cities/regions?" click. Both derive entirely from loaded reference data (continent field, `subdivisionsOf`), invent nothing, and scale because drill-down enumerates only the user's visited cities + a country's subdivisions, never the whole gazetteer.
- **IA & onboarding polish (#26, #24, #20).** Move Backup + Attribution to a dedicated "Data & sources" gear destination so Places does one job (#26); add a one-time first-run welcome card (#24); and connect the tabs so review turns into action (#20), fixing the inert `.city-focus` div in `VisitsList` along the way.
- **Design-system foundation (#29–#32).** Type/spacing tokens, unified visited/coverage color (fixing the black `#0b1220` city dot that mismatches the green toggle), a consistent SVG icon vocabulary, and a self-hosted OFL display face. Mostly maintainability + perceived-quality; sequence after the functional wins. Any font must be self-hosted (no Google Fonts CDN) to preserve the offline / no-proprietary-deps constraints.

*All 42 items are client-side only and were vetted against the code as compatible with the hard constraints (local-first/offline, no telemetry/accounts, no proprietary/Google deps, aggregator-only/inert data, keyboard-first + WCAG 2.1 AA, not a trip planner). Where a "how" cited a wrong line or over-claimed reuse (e.g. the two search ideas both need net-new keyboard logic; #21's country-frame needs a bbox derived from bundled Natural Earth geometry, not existing plumbing; #40's Welcome card doesn't exist yet), the merged entries above reflect the corrected scope.*
